# Publishing Example

Publishing is optional and begins only after explicit authorization.

## Export boundary

Select approved Wiki pages and public assets. Exclude Raw captures, credentials, private paths, internal logs, and unpublished drafts by default.

## Transform

Convert internal links and metadata to the target site's conventions. Keep the source wiki as the single editable knowledge source rather than editing exported copies independently.

## Validate

Build the site locally, check internal links and navigation, scan the export for sensitive information, and inspect representative desktop and mobile pages.

## Publish

Use the target repository's review and deployment process. Creating repositories, pushing changes, opening reviews, and deploying remain separate authorized actions.

## Verify

Confirm public routes and assets after deployment. If validation fails, keep the previous public version active and correct the source before exporting again.
