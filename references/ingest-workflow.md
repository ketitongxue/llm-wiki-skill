# Ingest Workflow

## 1. Orient

Read `purpose.md`, `SCHEMA.md`, `index.md`, the newest `log.md` entries, and relevant hubs. Confirm that the source belongs in scope.

## 2. Capture provenance

Create a Raw record containing a stable title, source identifier, capture date, and enough context to verify later claims. Store local attachments beside Raw records when authorized.

## 3. Extract reusable knowledge

Identify mechanisms, concepts, entities, constraints, decisions, and open questions. Separate direct claims from interpretation.

## 4. Reinforce before creating

Search for existing pages that express the same idea. Update them when the new source adds evidence, a constraint, or a conflicting view. Create a page only for a distinct reusable identity.

## 5. Connect

Link from the relevant hub to the durable page. Add reciprocal links when a reader benefits in both directions. Explain the relationship in surrounding prose.

## 6. Resolve conflicts

Preserve each sourced claim, state why they differ, and record the current synthesis without erasing uncertainty.

## 7. Synchronize

Update `index.md`, prepend a dated `log.md` entry, and run the lint checklist. Report changed pages, sources, unresolved conflicts, and any skipped checks.
